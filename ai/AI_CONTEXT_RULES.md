# AI Context Rules

The non-negotiable rules every agent must follow in this repo. If a rule conflicts with a user instruction, the rule wins and the user is informed.

## Trust hierarchy

1. **Repo source files** are ground truth.
2. **State files in this repo** (`docs/`, `memory/`, `ai/`, `PROJECT_CONTROL_LOG.md`, `INTEGRATION_NOTES.md`) are second truth. They describe the project, not the system.
3. **The chat transcript** is third truth. It is a working scratchpad, not a source of record.
4. **The model's training data** is fourth truth. It is unreliable for project specifics. Verify against the repo before acting on it.

If anything in tiers 2-4 conflicts with tier 1, update tiers 2-4 before stopping.
